  • Best Practices
    • Imports

      Separate dependency imports (ie external stuff) from local (stuff you write) by a newline

    • propTypes and defaultProps

      put them near the top

    • Methods

      should look like this and be next in the class

      handleSubmit = (e) => {
    e.preventDefault()
    this.props.model.save()
  }
  
  handleNameChange = (e) => {
    this.props.model.changeName(e.target.value)
  }
  
  handleExpand = (e) => {
    e.preventDefault()
    this.setState({ expanded: !this.state.expanded })
  }
    • setState

      this method is actually asynchronous SO

      you shouldn't rely on current state

      use prevState

      this.setState(prevState => ({ expanded: !prevState.expanded }))
    • destructing props

      should do it like this so you don't need to keep referencing this.props

      	render() {
    const {
      model,
      title
    } = this.props
    return ( 
      <ExpandableForm 
        onSubmit={this.handleSubmit} 
        expanded={this.state.expanded} 
        onExpand={this.handleExpand}>
        <div>
          <h1>{title}</h1>
          <input
            type="text"
            value={model.name} // notice the use of newline here
            onChange={this.handleNameChange}
            placeholder="Your Name"/>
        </div>
      </ExpandableForm>
    )
  }

How to write Great React

  • link
  • summary
    A component is just a function. We like it when functions are few lines and handle a minimal amount of responsibilities. If one function gets very long and confusing, break it up into multiple functions. Once upon a time, engineers had to keep variable and function names are short as possible because of memory constraints. Now, we are able to make the names as long and as descriptive as we desire. Still aim to keep the name short but highly specific / descriptive. Do not put the type of the information stored in the variable in the name. Do not include the bolded words in the following example variable names: userPreferencesObject userNameString moviesWatchedList
If I could five a new developer one piece of advice to hekp them write great React, what would it be?
  • Great React
    • goal:

      easy to read

      easy to maintain

      easy to extend

    • bulk of code is

      components

    • treat components as

      functions.

    • writing great functions:

      they are:

      1. small

        "should rarely be more than 20 lines"

        50 lines a good rule of thumb for react (render method)

      1. do one thing

        split your UI into small parts that do one thing

      1. have one level of abstraction

        not very many props

        3 is fine, 5 is too much, more than 7 is a crisis

        try to handle events at a lower level

      1. less than 3 arguments
      1. descriptive name
